# Интернет-магазин «Web-Larёk»
«Web-Larёk» — это интернет-магазин с товарами для веб-разработчиков, где пользователи могут просматривать товары, добавлять их в корзину и оформлять заказы. Сайт предоставляет удобный интерфейс с модальными окнами для просмотра деталей товаров, управления корзиной и выбора способа оплаты, обеспечивая полный цикл покупки с отправкой заказов на сервер.

## Архитектура приложения

Код приложения разделен на слои согласно парадигме MVP (Model-View-Presenter), которая обеспечивает четкое разделение ответственности между классами слоев Model и View. Каждый слой несет свой смысл и ответственность:

Model - слой данных, отвечает за хранение и изменение данных.  
View - слой представления, отвечает за отображение данных на странице.  
Presenter - презентер содержит основную логику приложения и  отвечает за связь представления и данных.

Взаимодействие между классами обеспечивается использованием событийно-ориентированного подхода. Модели и Представления генерируют события при изменении данных или взаимодействии пользователя с приложением, а Презентер обрабатывает эти события используя методы как Моделей, так и Представлений.

### Базовый код

#### Класс Component
Является базовым классом для всех компонентов интерфейса.
Класс является дженериком и принимает в переменной `T` тип данных, которые могут быть переданы в метод `render` для отображения.

Конструктор:  
`constructor(container: HTMLElement)` - принимает ссылку на DOM элемент за отображение, которого он отвечает.

Поля класса:  
`container: HTMLElement` - поле для хранения корневого DOM элемента компонента.

Методы класса:  
`render(data?: Partial<T>): HTMLElement` - Главный метод класса. Он принимает данные, которые необходимо отобразить в интерфейсе, записывает эти данные в поля класса и возвращает ссылку на DOM-элемент. Предполагается, что в классах, которые будут наследоваться от `Component` будут реализованы сеттеры для полей с данными, которые будут вызываться в момент вызова `render` и записывать данные в необходимые DOM элементы.  
`setImage(element: HTMLImageElement, src: string, alt?: string): void` - утилитарный метод для модификации DOM-элементов `<img>`


#### Класс Api
Содержит в себе базовую логику отправки запросов.

Конструктор:  
`constructor(baseUrl: string, options: RequestInit = {})` - В конструктор передается базовый адрес сервера и опциональный объект с заголовками запросов.

Поля класса:  
`baseUrl: string` - базовый адрес сервера  
`options: RequestInit` - объект с заголовками, которые будут использованы для запросов.

Методы:  
`get(uri: string): Promise<object>` - выполняет GET запрос на переданный в параметрах ендпоинт и возвращает промис с объектом, которым ответил сервер  
`post(uri: string, data: object, method: ApiPostMethods = 'POST'): Promise<object>` - принимает объект с данными, которые будут переданы в JSON в теле запроса, и отправляет эти данные на ендпоинт переданный как параметр при вызове метода. По умолчанию выполняется `POST` запрос, но метод запроса может быть переопределен заданием третьего параметра при вызове.  
`handleResponse(response: Response): Promise<object>` - защищенный метод проверяющий ответ сервера на корректность и возвращающий объект с данными полученный от сервера или отклоненный промис, в случае некорректных данных.

#### Класс EventEmitter
Брокер событий реализует паттерн "Наблюдатель", позволяющий отправлять события и подписываться на события, происходящие в системе. Класс используется для связи слоя данных и представления.

Конструктор класса не принимает параметров.

Поля класса:  
`_events: Map<string | RegExp, Set<Function>>)` -  хранит коллекцию подписок на события. Ключи коллекции - названия событий или регулярное выражение, значения - коллекция функций обработчиков, которые будут вызваны при срабатывании события.

Методы класса:  
`on<T extends object>(event: EventName, callback: (data: T) => void): void` - подписка на событие, принимает название события и функцию обработчик.  
`emit<T extends object>(event: string, data?: T): void` - инициализация события. При вызове события в метод передается название события и объект с данными, который будет использован как аргумент для вызова обработчика.  
`trigger<T extends object>(event: string, context?: Partial<T>): (data: T) => void` - возвращает функцию, при вызове которой инициализируется требуемое в параметрах событие с передачей в него данных из второго параметра.

### Данные

#### Интерфейс IProduct
Описывает структуру товара, применяется для классов ProductsCatalog и Cart.

Поля интерфейса:
`id: string` - уникальный идентификатор товара.

`description: string` - описание товара.

`image: string` - URL изображения товара.

`title: string` - название товара.

`category: string` - категория товара.

`price: number | null` - цена товара (может отсутствовать).

#### Интерфейс IBuyer
Хранит контактные данные и предпочтения оплаты покупателя при оформлении заказа, применяется для класса Buyer и использует тип данных TPayment.

Поля интерфейса:
`payment: TPayment` - способ оплаты.

`email: string` - email покупателя.

`phone: string` - телефон покупателя.

`address: string` - адрес доставки.

#### Тип данных TPayment
Перечисление допустимых способов оплаты покупателя.

`type TPayment = 'online' | 'cash'` - используем два способа оплаты.

#### Класс ProductCatalog
Управление полным списком товаров и детальной информацией о выбранном товаре.

Конструктор класса не принимает параметров.

Поля класса:
`items: IProduct[]` - массив всех доступных товаров.

`previewItem: IProduct | null` - товар, выбранный для подробного отображения.

Методы класса:
`setItems(products: IProduct[]): void` - сохранение массива товаров полученного в параметрах метода.

`getItems(): IProduct[]` - возвращает все товары из каталога.

`getItemById(id: string): IProduct | undefined` - находит товар по ID. Возвращает undefined, если товар не найден.

`setPreviewItem(item: IProduct): void` - сохранение товара для подробного отображения.

`getPreviewItem(): IProduct | null` - получение товара для подробного отображения.

#### Класс Cart
Хранение товаров, выбранных пользователем для покупки, и расчёт их стоимости.

Конструктор класса не принимает параметров.

Поля класса:
`items: IProduct[]` - массив товаров в корзине.

Методы класса:
`getItems(): IProduct[]` - возвращает текущий список товаров в корзине.

`addItem(item: IProduct): void` - добавляет товар в корзину.

`removeItem(item: IProduct): void` - удаляет товар из корзины.

`clear(): void` - очищает корзину.

`getTotal(): number` - возвращает общую стоимость товаров в корзине (сумма price, игнорируя null).

`getCount(): number` - возвращает количество товаров в корзине.

`hasItem(id: string): boolean` - проверяет наличие товара в корзине по ID.

#### Класс Buyer
Хранение и валидация контактных данных покупателя для оформления заказа.

Конструктор класса не принимает параметров.

Поля класса:
`payment: TPayment | null` - выбранный способ оплаты (по умолчанию null).

`email: string` - email покупателя (по умолчанию '').

`phone: string` - телефон покупателя (по умолчанию '').

`address: string` - адрес доставки (по умолчанию '').

Методы класса:
`set(field: string, value: string | TPayment): void` - универсальный сеттер для обновления любого поля. Параметры:
    `field` — название поля ('payment', 'email', 'phone', 'address'),
    `value` — новое значение.

`getData(): IBuyer` - возвращает объект с данными покупателя в формате IBuyer.

`clear(): void` - сбрасывает все поля к значениям по умолчанию.

`validate(): Record<string, string>` - проверяет заполненность полей. Возвращает объект ошибок, где:
    Ключ — название поля с ошибкой,
    Значение — текст ошибки.
Поля без ошибок отсутствуют в объекте. Пустой объект означает, что все данные валидны.

### Слой коммуникации

#### Класс ApiService
Класс отвечает за коммуникацию с API сервера "веб-ларёк". Он выполняет GET-запросы для получения данных о товарах и POST-запросы для отправки информации о заказах.

Поля класса:
`api: IApi` - экземпляр класса для работы с API сервера.

Конструктор класса:
`constructor(api: IApi)` - принимает объект для выполнения HTTP-запросов.

Методы класса:
`getProducts(): Promise<IProduct[]>` - выполняет GET-запрос на эндпоинт /product/ и возвращает массив товаров в формате `IProduct[]`.

`createOrder(order: IOrderRequest): Promise<IOrderResponse>` - выполняет POST-запрос на эндпоинт /order/ с данными о заказе. Использует `order: IOrderRequest` -  объект с данными о заказе и возвращает `Promise<IOrderResponse>` - промис с данными созданного заказа.

#### Интерфейс IProductsApiResponse
Описывает структуру ответа сервера при запросе полного списка товаров.

Поля интерфейса:
`total: number` - общее количество товаров в базе данных. Необходимо для пагинации и отображения информации о количестве доступных товаров.

`items: IProduct[]` - массив товаров, где каждый элемент соответствует интерфейсу IProduct. (см. в разделе "Данные")

#### Интерфейс IOrderRequest
Описывает структуру данных для создания заказа. Наследуется от IBuyer и расширяется информацией о заказе. (см. в разделе "Данные")

Поля интерфейса:
`total: number` - общая стоимость заказа.

`items: string[]` - массив ID товаров, которые входят в заказ. Каждый ID должен соответствовать существующему товару в базе.

#### Интерфейс IOrderResponse
Описывает структуру ответа сервера после успешного создания заказа.

Поля интерфейса:
`id: string` - уникальный идентификатор созданного заказа.

`total: number` - фактическая стоимость заказа, подтвержденная сервером.

#### Класс Header
Класс отвечает за отображение и управление компонентом шапки сайта. Он обрабатывает клики по кнопке корзины и отображает количество товаров в корзине.

`counterElement: HTMLElement` - элемент счетчика товаров в корзине.

Конструктор класса:

Методы класса:
`set counter(value: number)` - устанавливает значение счетчика товаров в корзине. Использует `value: number` - количество товаров и обновляет текстовое содержимое элемента счетчика.

#### Класс Gallery
Класс отвечает за отображение и управление компонентом галереи товаров. Он заменяет дочерние элементы контейнера галереи на переданный массив карточек.

Поля класса:
`catalogElement: HTMLElement` - контейнер элементов галереи товаров.

Конструктор класса:
`constructor(container: HTMLElement)` - принимает контейнер с элементами галереи.

Методы класса:
`set catalog(items: HTMLElement[])` - устанавливает массив элементов карточек товаров в галерею. Использует `items: HTMLElement[]` - массив элементов карточек и заменяет все дочерние элементы контейнера галереи на переданные элементы.

#### Классы карточек

##### Класс Card
Класс отвечает за отображение базового компонента карточки товара. Он устанавливает название и стоимость товара в соответствующие элементы карточки.

Поля класса:
`cardTitleElement: HTMLElement` - элемент отображения названия товара.

`cardPriceElement: HTMLElement` - элемент отображения стоимости товара.

Конструктор класса:
`constructor(container: HTMLElement)` - принимает контейнер с элементами карточки товара.

Методы класса:
`set cardTitle(value: string)` - устанавливает название товара в карточку. Использует `value: string` - название товара и обновляет текстовое содержимое элемента названия.

`set cardPrice(value: number | null)` - устанавливает стоимость товара в карточку. Использует `value: number | null` - стоимость товара в синапсах или `null`, обновляет текстовое содержимое элемента цены. Если значение равно `null`, отображает "Бесценно", иначе отображает стоимость в формате "{значение} синапсов".

##### Класс GalleryCard
Класс отвечает за отображение и управление компонентом карточки товара в галерее. Он наследует базовый функционал от класса `Card` и добавляет отображение изображения, категории и обработку кликов по карточке.

Поля класса:
`cardImageElement: HTMLImageElement` - элемент изображения товара.

`cardCategoryElement: HTMLElement` - элемент отображения категории товара.

`cardButton: HTMLButtonElement` - элемент кнопки карточки для обработки кликов.

Конструктор класса:
`constructor(events: IEvents, container: HTMLElement, onClick?: () => void)` - принимает объект событий для эмиссии событий, контейнер с элементами карточки и опциональный коллбэк `onClick` для обработки кликов по карточке.

Методы класса:
`set cardImage(item: HTMLImageElement)` - устанавливает элемент изображения товара. Использует `item: HTMLImageElement` - элемент изображения и сохраняет его в поле класса.

`set cardImageSrc(value: string)` - устанавливает источник изображения товара. Использует `value: string` - URL изображения и обновляет атрибут src элемента изображения.

`set cardCategory(value: string)` - устанавливает категорию товара. Использует `value: string` - название категории, устанавливает текстовое содержимое элемента категории, очищает все существующие классы категорий и добавляет соответствующий класс из маппинга `categoryMap` для визуального отображения категории.

##### Класс PreviewCard
Класс отвечает за отображение и управление компонентом карточки товара в режиме предпросмотра. Он наследует базовый функционал от класса `Card` и добавляет отображение изображения, категории, описания товара и кнопки добавления в корзину.

Поля класса:
`cardImageElement: HTMLImageElement` - элемент изображения товара.

`cardCategoryElement: HTMLElement` - элемент отображения категории товара.

`cardTextElement: HTMLElement` - элемент отображения описания товара.

`cardButton: HTMLButtonElement` - кнопка добавления удаления товара из корзины.

Конструктор класса:

### События

### События слоя представления

#### События модального окна

`modal:close` - событие закрытия модального окна. Генерируется при клике на кнопку закрытия модального окна в компоненте `Modal` и при клике на кнопку "За новыми покупками" в компоненте `Success`.

#### События корзины

`basket:open` - событие открытия корзины товаров. Генерируется при клике на кнопку корзины в компоненте `Header`.

`basket:clear` - событие очистки корзины. Генерируется при клике на кнопку "За новыми покупками" в компоненте `Success`. Приводит к очистке корзины и данных покупателя.

#### События форм

`orderForm:open` - событие открытия формы заказа. Генерируется при клике на кнопку оформления заказа в компоненте `Basket` (если кнопка не заблокирована).

`orderForm:field-changed` - событие изменения поля в форме заказа. Генерируется при клике на кнопки выбора способа оплаты или при вводе в поле адреса в компоненте `OrderForm`. Передает параметры `{ field: string, value: string }`, где `field` - имя поля ('payment' или 'address'), `value` - новое значение поля.

`contactsForm:open` - событие открытия формы контактов. Генерируется при клике на кнопку отправки в компоненте `OrderForm`.

`contactsForm:field-changed` - событие изменения поля в форме контактов. Генерируется при вводе в полях email или телефона в компоненте `ContactsForm`. Передает параметры `{ field: string, value: string }`, где `field` - имя поля ('email' или 'phone'), `value` - новое значение поля.

`success:open` - событие открытия экрана успеха. Генерируется при клике на кнопку отправки в компоненте `ContactsForm`.

#### События карточек
Компоненты `GalleryCard`, `PreviewCard` и `BasketCard` не генерируют событий напрямую, а вызывают коллбэки, переданные в конструкторе:

`GalleryCard` вызывает коллбэк `onClick` при клике на карточку, который вызывает `productCatalog.setPreview(item)` что генерирует событие `catalog:preview-changed`

`PreviewCard` вызывает коллбэк `onBasketToggle` при клике на кнопку корзины, который добавляет или удаляет товар из корзины через `cart.addItem(item)` или `cart.removeItem(item.id)`, что генерирует событие `cart:changed`

`BasketCard` вызывает коллбэк `deleteButton` при клике на кнопку удаления, который удаляет товар из корзины через `cart.removeItem(item.id)`, что генерирует событие `cart:changed`

### События слоя  моделей

`catalog:changed` - событие изменения каталога товаров. Генерируется моделью `ProductCatalog` при обновлении списка товаров. Передает параметры `{ items: IProduct[] }`.

`catalog:preview-changed` - событие установки превью товара. Генерируется моделью `ProductCatalog` при выборе товара для просмотра. Передает параметры `{ item: IProduct }`.

`cart:changed` - событие изменения корзины. Генерируется моделью `Cart` при добавлении или удалении товаров из корзины. Передает параметры `{ items: IProduct[], total: number, count: number }`, где `items` - массив товаров в корзине, `total` - общая стоимость, count - количество товаров.

`buyer:changed` - событие изменения данных покупателя. Генерируется моделью `Buyer` при обновлении данных (оплата, адрес, контакты) или при очистке данных. Передает параметры `{ data: Partial<IBuyer> }`.
